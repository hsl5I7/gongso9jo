# 앙상블 모델 설명 (XGBoost + LSTM)

> 처음 보는 사람도 한 번에 이해할 수 있도록 정리.
> 컨셉: **"두 모델의 예측 점수를 합쳐서 더 똑똑한 점수를 만든다"**

관련 스크립트: `ensemble_xgb_lstm.py`
출력 폴더: `ensemble_output/`

---

## 1) 입력 포맷

두 모델이 **각자 따로 만든 예측 파일**을 합칩니다.
조인 키는 `(date, ticker)` — 같은 날, 같은 종목.

| 출처 | 파일 경로 | 주요 컬럼 |
|------|----------|----------|
| **XGBoost** | `to_team_ensemble/predictions_all.parquet` | `date`, `ticker`, `p_pos` (상승확률), `label_binary` (정답), `test_year` |
| **LSTM** | `lstm/{general, general_yt}/lookback_60/fold_*/test_predictions.parquet` | `date`, `ticker`, `prob_익절` (상승확률), `label_binary`, fold 폴더명에서 `test_year` 추출 |

처리 흐름:
1. LSTM은 fold별로 흩어져 있어 **모든 fold를 concat**
2. XGB와 LSTM을 `(date, ticker)` 기준 **inner join**
3. **겹치는 8개 연도(2018~2025)** 의 행만 분석 대상
4. 두 모델의 `label_binary`가 일치하는지 검증 후, XGB의 label을 정답으로 사용

---

## 2) 사용 모델 (앙상블 공식)

앙상블은 **새로운 학습 모델이 아님**. 두 확률을 섞는 **공식 4종**.

| 이름 | 공식 | 설명 |
|------|------|------|
| `p_xgb`, `p_lstm` | 원본 | 비교용 베이스라인 |
| `p_avg` | `(p_xgb + p_lstm) / 2` | 단순 평균 |
| `p_avg_rank` | `(rank(p_xgb) + rank(p_lstm)) / 2` *(연도별 백분위)* | 두 모델 점수 스케일이 다르므로 **순위로 바꿔서** 평균 |
| `p_w{XX}xgb` | `w · p_xgb + (1 - w) · p_lstm`, w ∈ {0.2, 0.3, ..., 0.8} | XGB 가중치 그리드 탐색 |

> **왜 rank 평균이 필요한가?**
> XGB는 0.3~0.7 부근, LSTM은 0.1~0.9 등 분포가 달라서 그냥 평균 내면 한쪽이 과대 반영될 수 있음. 그래서 같은 연도 내에서 **순위(0~1)** 로 바꿔서 합산.

### ⚠️ 4가지 공식은 "비교용"이며, 최종 1개를 골라 운영

현재 스크립트는 **벤치마크 역할**. 한 번 실행하면 위 공식들이 모두 계산되어 10개 컬럼(`p_avg`, `p_avg_rank`, `p_w20xgb`~`p_w80xgb` + 원본 `p_xgb`, `p_lstm`)이 동시에 저장됨. 자동으로 하나를 선택하지 않음.

운영 흐름:
1. `metrics_{variant}.csv` 결과 확인
2. **AUC 또는 `win@0.65` 등 목표 지표가 가장 높은 공식 1개**를 사람이 선택
3. 그 공식만 실제 매수 신호로 사용

> 참고: `p_w50xgb`는 사실상 `p_avg`와 동일 (0.5·xgb + 0.5·lstm = 평균).

## 3) 출력 결과

앙상블 모델의 출력은 **"특정 날짜 · 특정 종목이 익절(목표 수익)에 도달할 확률"** 입니다.

### 출력의 형태 (행 단위)

`(date, ticker)` 한 쌍마다 **0~1 사이의 확률값 1개**가 나옵니다.

| date | ticker | p_ensemble |
|------|--------|-----------|
| 2024-03-12 | 005930 (삼성전자) | 0.72 |
| 2024-03-12 | 000660 (SK하이닉스) | 0.41 |
| 2024-03-12 | 035420 (NAVER) | 0.58 |
| ... | ... | ... |

> `p_ensemble`은 운영 시 선택한 공식 1개의 결과 (예: `p_avg_rank` 또는 `p_w60xgb`).

### 확률의 의미

- **1에 가까울수록** → 익절할 가능성이 높다고 모델이 판단
- **0에 가까울수록** → 손절 가능성이 높다고 판단
- 예: `p = 0.72` → 모델이 보기에 **72% 확률로 익절** 할 것으로 예상되는 종목

### 이 확률을 실전에서 어떻게 쓰는가

확률값 자체로는 매수/매도 결정이 안 됨. **임계값(threshold)** 을 정해서 필터링:

```
if p_ensemble >= 0.65:
    → 매수 후보 종목으로 선정
else:
    → 무시
```

임계값이 높을수록 → **선택 종목 수는 적지만 적중률(win rate)은 높음**.
임계값이 낮을수록 → **종목 수는 많지만 적중률은 낮음**.

이 trade-off를 `metrics_{variant}.csv`의 `n@0.65`, `win@0.65` 등으로 확인.

### 핵심 비교 포인트

- 단순평균 vs rank평균 vs 가중평균 중 **어떤 합성이 AUC와 win@ 모두 가장 높은가**
- 변종 `general` vs `general_yt` 중 어느 LSTM이 XGB와 더 좋은 시너지를 내는가

## (부록) 결과 파일

`ensemble_output/` 폴더에 **두 LSTM 변종(`general`, `general_yt`)별로 3개 파일씩**, 총 6개.

| 파일 | 내용 |
|------|------|
| `ensemble_{variant}.parquet` | 행별 확률값 (10개 공식 모두) |
| `metrics_{variant}.csv` | 전체 성능 요약: AUC, AP, 임계값별 `n@`/`win@` |
| `per_fold_auc_{variant}.csv` | 연도(fold)별 AUC 비교표 |

**평가 지표**:
- **AUC** — 모델이 정답을 얼마나 잘 변별하는가 (1에 가까울수록 좋음)
- **AP** — 양성(익절) 클래스에 대한 평균 정밀도
- **`n@0.65`** — 확률 ≥ 0.65인 종목 수 (매수 후보)
- **`win@0.65`** — 그 중 실제로 익절한 비율 (실전 적중률)
