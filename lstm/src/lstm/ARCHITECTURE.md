# LSTM 모델 아키텍처

## 한 줄 요약

**종목의 최근 N일치 시계열 데이터를 보고 지금 매수하면 익절할 확률을 0~1 사이 숫자로 예측하는 모델.**


- **입력**: 한 종목의 최근 30~120일치 피처(가격·거래량·기술지표 등)
- **출력**: 익절 확률 `P(익절) ∈ [0, 1]`
- **라벨**: 진입가 기준 손절선(−2 ATR)보다 익절선(Chandelier +3 ATR)에 먼저 닿으면 1, 아니면 0

## 모델 구조

   [입력 시퀀스]
        │
        ▼
   ┌─────────────┐
   │    LSTM     │     시계열의 순서·맥락을 학습하는 RNN의 한 종류
   │  2-layer    │     → 과거 N일을 거치며 "현재 상태"를 128차원 벡터로 압축
   │  hidden=128 │
   └─────────────┘
        │
        ▼ (마지막 시점 벡터만 사용)
   ┌─────────────┐
   │    MLP      │     Dropout → Linear(128→64) → ReLU → Linear(64→1)
   │   Head      │     → 압축된 벡터를 단일 점수로 변환
   └─────────────┘
        │
        ▼ sigmoid
   ┌─────────────┐
   │  P(익절)    │     0~1 사이의 확률값
   └─────────────┘
\\\

## 학습 방식

| 항목 | 설정 | 한 줄 설명 |
|---|---|---|
| **Loss** | BCEWithLogitsLoss | 이진분류 표준 손실함수 |
| **Optimizer** | AdamW (lr=1e-4) | 가중치 감쇠 포함 적응형 옵티마이저 |
| **과적합 방지** | Dropout + Gradient Clipping + Early Stopping | 일반화 성능 확보 |
| **입력 정규화** | 같은 날짜 종목들 간 z-score | 시장 분위기에 자동 적응 |
| **검증 방법** | Walk-forward (14 fold) | 과거로 학습 → 미래로 검증, 시간 누설 차단 |


- **왜 LSTM?** — 주가는 시계열이므로 순서 정보가 중요. LSTM은 긴 시퀀스의 의존 관계를 학습 가능
- **왜 마지막 시점만?** — 지금 진입할지 결정하는 문제이므로 현재 시점의 표현만 필요
- **왜 Walk-forward?** — 미래 데이터로 과거를 학습하면 안 됨 → 시간 순서대로 fold 분할 + 60일 gap으로 라벨 누설 방지
- **왜 z-score 정규화?** — 종목별 절대값이 아닌 **그날 시장 안에서의 상대적 위치**로 학습 → 강세장/약세장 변동에 강건
